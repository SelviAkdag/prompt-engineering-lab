What is Prompt Engineering?

Prompt Engineering is the art and science of designing effective instructions (prompts) for language models to achieve the best possible results. It involves understanding how models "think" and using the right techniques to guide them toward correct answers.

üéØ Why is it important?

  • Better results: Well-designed prompts can dramatically improve model performance
  • Cost efficiency: Good prompts can achieve better results with smaller/cheaper models
  • Reliability: Consistent prompts lead to more predictable and reliable outputs
  • Task-specific optimization: Different tasks require different prompting strategies

üìò What you will learn

You will explore how large language models handle sentiment classification ‚Äì deciding whether a movie review expresses a positive or negative emotion. Throughout the notebook, you will apply and compare the following prompting techniques:

  • Zero-shot ‚Äì asking the model directly without any examples.
  • Few-shot ‚Äì providing a few labeled examples before the task.
  • Chain-of-Thought (CoT) ‚Äì encouraging the model to reason step-by-step before answering.
  • Self-consistency ‚Äì running multiple reasoning paths and choosing the most consistent outcome.

üß© About the dataset

The dataset consists of short movie reviews paired with sentiment labels (positive or negative). It's designed to be small and easy to experiment with ‚Äì perfect for testing how different prompting strategies affect model reasoning and performance.

üéØ Your tasks

  1. Understand the dataset: Explore the movie reviews and think about what linguistic cues signal positivity or negativity.
  2. Run and compare the methods: Test zero-shot, few-shot, CoT, and self-consistency prompts and observe how accuracy and reasoning differ.
  3. Experiment interactively: Try writing your own example reviews or related questions to see how the model responds under each method.
  4. Reflect and analyze: Consider which approach generalizes best, and why.

1.2 Local Model Setup (Required)

‚ö†Ô∏è Important: This lesson requires a running LLM. Choose Ollama (free, local) or OpenAI API (paid).

Option 1: Ollama (Recommended) ‚úÖ

Ollama runs models locally on your computer - completely free and private.

  1. Install Ollama:
    Download from https://ollama.ai/download
    Or via package manager:
    • Windows: winget install Ollama.Ollama
    • Mac: brew install ollama
    • Linux: curl -fsSL https://ollama.ai/install.sh | sh
  2. Add Ollama to PATH (Windows):
    If ollama is not recognized in terminal:
    1. Open "Edit system environment variables" (search in Start menu)
    2. Click "Environment Variables..." at the bottom
    3. Under "System variables", find and double-click "Path"
    4. Click "New" and add: C:\Users\YourUsername\AppData\Local\Programs\Ollama
    5. Click OK on all windows
    6. Restart VS Code/Cursor for PATH to update
  3. Download the model:
    ollama pull llama3.2:3b
    (Downloads ~2GB, takes a few minutes)
  4. Create extended token model:
    ollama create llama3.2-long -f Modelfile
    (This creates a custom model with 300 token limit instead of default 128)
    Note: The Modelfile was auto-created during setup.
  5. Verify installation:
    ollama list
    You should see both llama3.2:3b and llama3.2-long in the list.
  6. Verify .env configuration:
    Make sure your .env file contains:
    MODEL_BACKEND=ollama
    OLLAMA_MODEL=llama3.2-long

Option 2: OpenAI API (Alternative) üí≥

Use OpenAI if you prefer cloud-based inference (costs ~$0.01-0.05 per lesson run).

  1. Get API key: https://platform.openai.com/api-keys
  2. Set environment variables in .env:
    MODEL_BACKEND=openai
    OPENAI_API_KEY=sk-your-key-here
Recommendation: Use Ollama for this lesson. It's free, fast enough, and you don't need to manage API keys or worry about costs.
